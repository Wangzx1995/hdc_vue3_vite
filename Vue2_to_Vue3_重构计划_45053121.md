# Vue 2 到 Vue 3 重构计划

## 项目概况
- 当前：Vue 2.6.10 + Webpack 3 + Element UI + iview + hui
- 目标：Vue 3.4+ + Vite 5 + Element Plus + 自定义适配组件
- 文件规模：373+ 个 .vue 文件
- 策略：**逐步迁移**，保留 Options API，最小化业务代码改动

---

## Task 1：搭建 Vue 3 + Vite 基础框架

### 1.1 创建 Vite 配置文件
- 新建 `vite.config.js`，配置路径别名 `@` → `src`
- 配置 SCSS/Less 预处理器支持
- 配置代理（从 `config/index.js` 迁移 devServer.proxy）
- 配置静态资源目录 `public/`

### 1.2 更新 package.json 依赖
**移除（Webpack/Babel 6 相关）**：
- webpack 3.x 全家桶、babel 6.x 全家桶、vue-loader、vue-template-compiler
- extract-text-webpack-plugin、optimize-css-assets-webpack-plugin 等

**升级/新增**：
- `vue` → `^3.4.0`
- `vite` → `^5.0.0`
- `@vitejs/plugin-vue` → `^5.0.0`
- `vue-router` → `^4.2.0`
- `vuex` → `^4.1.0`
- `vue-i18n` → `^9.0.0`
- `element-plus` → `^2.5.0`
- `@element-plus/icons-vue` → 图标库

### 1.3 迁移 HTML 入口
- 将 `index.html` 改为 Vite 入口，引入 `<script type="module" src="/src/main.js">`

### 1.4 环境变量迁移
- 将 `config/dev.env.js`、`config/prod.env.js` 迁移到 `.env.development`、`.env.production`
- 全局替换 `process.env` → `import.meta.env`

---

## Task 2：迁移核心入口文件

### 2.1 迁移 `src/main.js`
**Vue 2 风格**：
```js
import Vue from "vue/dist/vue.esm.js";
new Vue({ el: "#app", router, store, template: "<App/>", components: { App } });
```
**Vue 3 风格**：
```js
import { createApp } from "vue";
const app = createApp(App);
app.use(router).use(store).use(i18n).mount("#app");
```

### 2.2 全局属性迁移
将 `Vue.prototype.$xxx` 替换为 `app.config.globalProperties.$xxx`：
- `$moment`、`$echarts`、`$deepCopy`、`$bas`
- 提供一个全局兼容插件，将 `getCurrentInstance().proxy` 封装为类似 Vue 2 的 `this`

### 2.3 全局组件注册
- `Vue.component()` → `app.component()`
- `Vue.use()` → `app.use()`
- `Vue.config.productionTip` → 移除（Vue 3 无此选项）

---

## Task 3：迁移路由 `src/router/index.js`

### 3.1 路由创建方式
- `new Router()` → `createRouter({ history: createWebHashHistory(), routes })`
- 删除 `Vue.use(Router)`

### 3.2 移除 Router.prototype.push hack
Vue Router 4 已修复重复导航的报错，删除以下代码：
```js
const originalPush = Router.prototype.push;
Router.prototype.push = function push(location, resolve, reject) { ... }
```

### 3.3 动态导入适配
当前使用 `_import` 函数配合 webpack require：
```js
const _import = require("./_import_" + process.env.NODE_ENV);
```
Vite 下改为原生 `import()` 或 `import.meta.glob`：
```js
const modules = import.meta.glob("../views/**/*.vue");
const _import = (path) => modules[`../views/${path}.vue`];
```

---

## Task 4：迁移状态管理 `src/store/`

### 4.1 创建 Store 方式
- `new Vuex.Store()` → `createStore()`
- 删除 `Vue.use(Vuex)`

### 4.2 Store 模块兼容性
Vuex 4 的 module 语法与 Vuex 3 基本一致，现有 7 个 store module（app、user、permission、tree、configView、exportManage、downloadManage）只需调整创建方式即可。

---

## Task 5：UI 库迁移

### 5.1 Element UI → Element Plus
- 全量引入：`import ElementPlus from "element-plus"` + `app.use(ElementPlus)`
- 样式：`import "element-plus/dist/index.css"`
- 迁移 `src/moudulesConfig/element.js` 中的配置到 Element Plus
- **注意**：Element Plus 的图标系统从字体图标改为 SVG，需要批量替换 `<i class="el-icon-xxx">` → `<el-icon><XXX /></el-icon>`

### 5.2 iview 组件替换（仅 3 个组件）
iview 无 Vue 3 版本，替换方案：
| iview 组件 | Element Plus 替代 |
|-----------|------------------|
| `<Icon type="xxx">` | `<el-icon><XXX /></el-icon>` |
| `<Poptip>` | `<el-popover>` |
| `<Upload>` | 已有 Element Plus Upload，统一替换 |

搜索范围：`src/iview.js` + 全局模板中使用 `Icon`、`Poptip`、`Upload` 的地方。

### 5.3 hui / @hui-pro 组件适配
项目中有以下 hui-pro 组件源码位于 `src/components/hui-pro/`，需要逐个适配到 Vue 3：
- `ellipsis`
- `holiday-picker`
- `plan`（含 plan-group、plan-point、plan-range 等子组件）
- `tree-select`
- `utils`

此外还有从 `hui` 包导入的 `Switch`、`IpInput`。

**适配策略**：
1. 先检查这些组件的 Vue 3 兼容性（大部分 Options API 组件在 Vue 3 下可直接运行）
2. 对于使用了 `Vue.extend`、`Vue.component` 等全局 API 的组件，替换为 `defineComponent`
3. 对于使用了自定义指令内部机制的组件，按 Vue 3 指令 API 重写

---

## Task 6：全局代码兼容层

### 6.1 过滤器（Filters）迁移
Vue 3 已移除过滤器，当前 `src/filters/` 下的过滤器需要改为：
- 全局方法挂载到 `app.config.globalProperties`
- 或在组件内改为 `computed` / `methods`

### 6.2 自定义指令迁移
`src/directives/index.js` 中的指令需要按 Vue 3 指令钩子调整：
- `bind` → `mounted`
- `inserted` → `mounted`
- `update` → `updated`
- `componentUpdated` → `updated`
- `unbind` → `unmounted`

### 6.3 Vue 2 全局 API 移除
搜索并替换以下全局 API：
- `Vue.set(target, key, value)` → 直接 `target[key] = value`
- `Vue.delete(target, key)` → `delete target[key]`
- `Vue.observable(obj)` → `reactive(obj)`（需从 vue 导入）
- `Vue.nextTick()` → `nextTick()`（需从 vue 导入）

### 6.4 $listeners 合并到 $attrs
Vue 3 中 `$listeners` 已移除，所有事件监听器都通过 `$attrs` 传递。需要检查使用了 `$listeners` 的组件（如封装通用组件时）。

## Task 7：其他依赖兼容性处理

### 7.1 需要检查/替换的依赖
| 依赖 | Vue 2 版本 | Vue 3 处理方案 |
|------|-----------|---------------|
| `vue-baidu-map` | 0.21.22 | 需替换为 Vue 3 兼容版本或自研 |
| `v-charts` | 1.16.12 | 基于 Vue 2，需改用 ECharts 直接封装 |
| `v-viewer` | 1.6.4 | 检查是否有 Vue 3 版本（viewerjs 本身无 Vue 依赖） |
| `vue-clipboard2` | 0.3.1 | 可替换为原生 Clipboard API |
| `vue-scrollto` | 2.11.0 | 检查 Vue 3 兼容性 |
| `@vue/babel-helper-vue-jsx-merge-props` | 1.2.1 | Vue 3 JSX 语法不同，如使用 JSX 需升级 babel 插件 |

### 7.2 jQuery 相关代码
项目中存在 `jquery` 依赖，建议逐步移除，但在迁移阶段可保留。

## Task 8：逐步验证与测试

### 8.1 验证清单（按优先级）
1. **登录页** → 确保路由、状态管理、表单验证正常
2. **首页 Dashboard** → 确保图表、数据展示正常
3. **核心模块** → maintenanceCenter、manageCenter 等各主模块
4. **公共组件** → 侧边栏、头部、表格、表单等

### 8.2 常见问题排查
- 白屏：检查 Vite 构建错误、组件导入路径
- 样式丢失：检查 Element Plus 样式是否正确引入
- 图标不显示：检查 Element Plus 图标迁移
- API 请求异常：检查代理配置、baseURL

---

## 风险点与注意事项

1. **hui-pro 组件适配**：这些组件的复杂度未知，可能需要较多时间逐个调试
2. **iview 组件替换**：虽然只有 3 个组件，但如果在模板中被大量使用，替换工作量也会增加
3. **ECharts / v-charts**：v-charts 是 Vue 2 封装，建议直接改用 ECharts 5 的 API
4. **动态路由权限**：`src/store/modules/permission.js` 中的路由过滤逻辑需要重点验证
5. **Vite 的 `import.meta.glob`**：替换 webpack 的 `require.context` 时需要注意异步加载的兼容性

---

## 实施建议

1. **先建分支**：从当前 Vue 2 代码切出 `feature/vue3-migration` 分支
2. **Task 1-4 一起做**：基础环境、入口、路由、状态管理可以先集中完成，让项目能跑起来
3. **Task 5-7 按模块拆分**：每个模块单独测试，避免一次改动过多导致问题难以定位
4. **保留原构建配置**：在迁移完成前，不要删除 `build/` 和 `config/` 目录，可作为对比参考
